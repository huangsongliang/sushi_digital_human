# 2026-05-22

## 登录注册功能全量改进

### 后端改动
- **config.py**: 新增 `frontend_url`、`login_max_attempts`、`login_lockout_minutes` 配置项
- **auth.py**:
  - 添加 `_validate_password_strength()` — 密码强度校验（8位+大小写字母+数字+特殊字符）
  - `RegisterRequest` / `PhoneRegisterRequest` 使用 field_validator 校验密码强度
  - 添加登录限流：`_check_login_rate_limit()` + `_record_login_attempt()` 基于 Redis，默认 5 次/15 分钟
  - 登录/注册/手机登录/手机注册四个端点均加上限流检查
  - GitHub 回调前端 URL 从硬编码改为 `settings.frontend_url`

### 前端改动
- **新建 `frontend/.env`**: 添加 `VITE_API_BASE_URL=http://localhost:8000`
- **新建 `frontend/src/utils/request.ts`**: 集中式 HTTP 客户端
  - Token 自动刷新：401 时自动调用 /api/auth/refresh，并发请求只刷新一次
  - 刷新失败时触发 `auth:logout` 自定义事件
  - 统一错误处理：抛出 ApiError(status, message)
- **重写 `auth.ts` store**: 所有函数改用集中式 request 客户端，去除重复的 fetch 代码
- **重写 `AuthPage.vue`**:
  - 字段级表单校验（邮箱格式、手机号格式 /^1[3-9]\d{9}$/、密码强度可视化进度条）
  - 密码显示/隐藏切换按钮
  - 注册成功后绿色提示框 + 3 秒自动切换登录 tab
  - 两个独立 SMS 倒计时 timer（login/register 不再共用一个）
  - GitHub 登录使用 `import.meta.env.VITE_API_BASE_URL`
- **更新 `router/index.ts`**: 监听 `auth:logout` 事件自动跳转登录页
- **更新 `GithubCallback.vue`**: 增加错误提示文本

## Bug 修复 — 页面空白
- **问题**: `request.ts` 中的 `clearAuthTokens` 函数缺少 `export`，导致 `auth.ts` store 导入失败，整个 Vue 应用崩溃白屏
- **修复**: 给 `clearAuthTokens` 添加 `export` 关键字

## CI/CD 全面改造（18 个文件）

### 决策
- 部署目标：Kubernetes + Helm
- AI 自动修复：DashScope 优先，GitHub Models 兜底
- Nginx + 前端分发：方案 A（Dockerfile 包含 dist，init container 复制）
- 健康检查：保留后端 JSON 格式，CI 用 `curl -fsS` 依赖 HTTP 状态码
- nginx-ingress + cert-manager 暂不安装（Ingress 默认关闭）

### 重写/修改的文件
- **`.github/workflows/ci-cd.yml`** — 全面重写
  - 用 `astral-sh/setup-uv@v5` 替代手动安装 uv
  - `docker-compose` → `docker compose`
  - 新增 `frontend-lint` 和 `frontend-build` job（与大 CI 合并）
  - 新增 `helm-deploy` job（K8s 部署，需 `KUBE_CONFIG` secret）
  - 新增 `build-and-push` job 下载 frontend-dist artifact 后构建
  - `compose-test` 健康检查改用 `curl -fsS`
- **`Dockerfile`** production stage 新增 `COPY frontend/dist/ ./frontend/dist/`
