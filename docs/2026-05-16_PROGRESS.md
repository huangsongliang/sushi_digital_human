**核心配置 (`docker-compose.prod.yml`)：**
```yaml
services:
  api:
    image: sushi_digital_human-api
    build: .
    deploy:
      replicas: 3
    environment:
      - UVICORN_WORKERS=4
      - REDIS_URL=redis://redis:6379/0
      - DASHSCOPE_API_KEY=${DASHSCOPE_API_KEY}
      - HF_ENDPOINT=${HF_ENDPOINT:-https://hf-mirror.com}
    dns:
      - 8.8.8.8
      - 1.1.1.1
```

**解决方案**：

```python
import platform

is_windows = platform.system() == 'Windows'

success_icon = '✅' if not is_windows else '[OK]'
error_icon = '❌' if not is_windows else '[ERR]'
warning_icon = '⚠️' if not is_windows else '[WARN]'
```

**验证**：Windows 终端显示正常 ASCII 字符，文件日志保留 emoji

#### 4. 遗留问题修复：RAG 链与混合检索集成

**问题描述**：混合检索器未接入问答流程

**解决方案**：
- 修改 `rag_chain.py`，使用 `HybridRetriever` 替代原有检索器
- 确保流式输出与混合检索的兼容性
- 验证检索结果正确注入提示词

**核心代码**：
```python
from backend.retrieval.hybrid_retriever import HybridRetriever

class RAGChain:
    def __init__(self):
        self.retriever = HybridRetriever()

    async def generate(self, query: str, session_id: str):
        # 使用混合检索获取文档
        docs = await self.retriever.retrieve(query)
        # 构建提示词
        prompt = self._build_prompt(query, docs)
        # 调用 LLM 生成回答
        return await self.llm.generate(prompt)
```

**验证**：混合检索结果正确注入，回答质量提升

### 解决方案

#### 问题1：DNS 解析失败

**错误信息：**
```
Failed to resolve 'files.pythonhosted.org'
```

**原因**：Docker 容器内 DNS 配置异常

**解决方案**：
- 在 `docker-compose.prod.yml` 中添加 DNS 配置：
  ```yaml
  dns:
    - 8.8.8.8
    - 1.1.1.1
  ```

#### 问题2：PyPI SSL 错误

**错误信息：**
```
SSLError: [SSL] record layer failure
```

**原因**：阿里云镜像源不稳定

**解决方案**：
- 改用清华镜像源：
  ```dockerfile
  RUN pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
  ```

#### 问题3：pydantic 版本冲突

**错误信息：**
```
langchain 0.3.0 depends on pydantic<3.0.0 and >=2.7.4
```

**原因**：requirements.txt 中硬编码了旧版本

**解决方案**：
- 使用灵活版本要求：
  ```
  pydantic>=2.7.0
  pydantic-settings>=2.4.0
  ```

#### 问题4：torch==2.2.0+cpu 不存在

**错误信息：**
```
No matching distribution found for torch==2.2.0+cpu
```

**原因**：清华镜像源不提供 +cpu 后缀版本

**解决方案**：
- 改用 `torch>=2.0.0`，让 pip 自动选择合适版本

#### 问题5：容器名称冲突

**错误信息：**
```
The container name "/sushi-redis" is already in use
```

**原因**：之前的容器未正确清理

**解决方案**：
```bash
docker rm -f sushi-redis
```

#### 问题6：端口 80 被占用

**错误信息**：浏览器访问 `http://localhost` 返回空白或错误

**原因**：`com.docker.backend.exe` 占用了端口 80

**解决方案**：
- 将 Nginx 端口从 80 改为 8080：
  ```yaml
  ports:
    - "8080:80"
  ```

#### 问题7：前端空白页面

**错误信息**：浏览器控制台显示 `@vite/client` 加载失败

**原因**：前端使用开发模式构建，而非生产模式

**解决方案**：
```bash
cd frontend
npm run build
```
